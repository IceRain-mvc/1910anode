# *1、封装 面向对象三大特征之一

  • 什么是封装：将类的某些信息（属性或方法）隐藏在类内部，不许外部程序直接访问，
  • 而是通过该类提供的方法（getter/setter）来实现对隐藏信的操作和访问

# *2、封装的好处

  • 便于使用者正确使用系统，防止错误修改属性
  • 有助于系统之间的松耦合，提高系统独立性
  • 提高软件的可重用性
  • 降低了构建大型系统的风险

# 3、封装的步骤

  • 修改属性的可见性
  • 创建公有的getter/setter方法
  • 在getter/setter方法中加入属控制语句

注意：允许只出现一个getter或setter方法，只有一个getter代表当前属性只读，只有一个setter代表当前属性只写

# 4、包的作用

  • a、允许类组成较小的单元（类似文件夹），易于找到和使用相应的文件
  • b、有助于实施访问权限控制
  • c、防止命名冲突，区分名字相同的类

# 5、包命名规范

  • a、包名由小写字母组成，不能以圆点开头或结尾
  • b、包名之前最好加上唯一的前缀，通常使用组织倒置的网络域名
  • c、包名后续部分依不同机构内部的规范不同而不同

# 6、使用包的注意事项

  • a、一个类同时引用了两个来自不同包的同名类---必须通过完整类名来区分
  • b、每个包都是独立的，顶层包不会包含子包的类
  • c、package和import的顺序是固定的
    • package必须位于第一行（忽略注释行）
    • 只允许有一个package语句
    • 其次是import
    • 接着是类的声明

# *8、 static修饰符

# 8.1 static可以用来修饰:

  • 成员变量
    • 静态变量，可以直接通过类名访问 static int age;
  • 成员方法
    • 静态方法，可以直接通过类名访问 public static void show(){}

    注意：static修饰的方法中不能使用this和super关键字调用属性和方法（需要通过new对象来调用）

  • 代码块

    • *静态代码块，当Java虚拟机加载类时，就会执行该代码块 static{}

    • 实例成员和静态成员的区别： static修饰的成员特点，程序启动时立即分配空间，并且始终保持同一个空间，程序停止在释放空间，全局只创建一次。 实例成员（new对象才能用的）创建对象并调用的时候才会分配空间，方法执行完成就释放空间，可以被创建多次。

    • 什么时候使用static修饰符 1、static修饰的方法：制作工具类（处理字符串的，处理发送消息的，导入导出excel，加密的）中的方法 2、staitc修饰的属性：在多个类中共享同一个数据 3、static静态代码块：完成单例模式和程序的初始化工作

# 8.2 static修饰与非static修饰的区别

属性 类属性、类变量 实例属性、实例变量

# *10、继承

# * a、什么是继承：面向对象三大特征之一，指一个对象直接使用另一对象的属性和方法
# * b、为什么需要继承：减少代码冗余，方便维护
# * c、继承的特性
  • 子类可以调用父类非private的属性和方法。
    • 子类可以对父类的属性和方法进行扩展。
  • 子类可以用自己的方式实现父类的方法。
  • Java的继承是单继承，但是可以多重继承，
    • 单继承就是一个子类只能继承一个父类，
    • 多重继承就是，例如A类继承B类，B类继承C类，所以按照关系就是C类是B类的父类，B类是A类的父类
  • 提高了类之间的耦合性（继承的缺点，耦合度高就会造成代码之间的联系更紧密）。
# * d、继承关键字
  • 继承可以使用 extends 和 implements 这两个关键字来实现继承，而且所有的类都是继承于 java.lang.Object，
  • 当一个类没有继承的两个关键字，则默认继承Object
# * e、super关键字
  • 调用父类的属性、方法、构造方法
  • super只能出现在子类的方法和构造方法中
  • super调用构造方法时，只能是第一句
  • super不能访问父类的private成员
# * f、final修饰符
  • 修饰常量，声明时必须赋值
  • 修饰类，此类不能被继承--最终类
  • 修饰方法，方法不能被重写--最终方法
  • 修饰属性，属性不能被赋值。
# * g、访问修饰符protected
  • protected 意为受保护的，
  • 可以修饰属性和方法
  • 本类、同包、子类可以访问
# * h、继承条件下构造方法的调用规则
  • 子类构造方法默认调用父类的无参构造方法 super();
  • 子类构造方法通过super调用父类的有参构造方法，父类执行相应构造方法，而不执行父类无参构造方法 super("张三",20);
# * i、不能被继承的情况：
  • 1、private 成员
  • 2、子类与父类不在同包，使用默认访问权限的成员。
  • 3、构造方法
