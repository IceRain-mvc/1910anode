# *1、方法有四种：

  • 1、无参数无返回值:
public void 方法名(){}
1

什么时候用： 输出各种信息

  • 2、无参数有返回值:
public 数据类型  方法名(){ return 和返回类型，类型一样的结果;}
1

什么时候用： 想获取某个结果的时候

  • 3、有参数无返回值:
public void 方法名(数据类型 变量名,数据类型 变量名,...){}
1

什么时候用： 想给某个变量设置值的时候

  • 4、有参数有返回值:
public 数据类型   方法名(数据类型 变量名,数据类型 变量名,...){ return 和返回类型，类型一样的结果; }
1

什么时候用： 登陆，注册，支付，验证，加工数据

# 2、方法定义的一般形式

<访问修饰符> 返回类型 <方法名>(<参数列表>) { //方法的主体 }

注意： 如果无返回类型使用void ，如果有返回类型，必须使用return返回相应的类型

# 3、在同一个类中，方法之间可以相互调用，不需要创建对象来引用

# 4、什么时候需要给方法写返回值

1、 后续程序需要用到返回值时就应该写 int num= input.nextInt(); 2、 方法执行过程中需要返回处理后的结果时，则需要返回值 3、 保持调用者和被调用者之间的通信

# 5、注意：

形参是在定义方法时对参数的称呼（带有数据类型的参数） 实参是在调用方法时传递给方法的实际的值（带有实际值的参数） 调用带参方法时要求实参与形参要匹配

# *6、构造方法：

  • 特点： 与类同名 ,没有返回类型，默认隐藏，可以重载
  • 作用：完成对象的初始化工作（给属性赋值）
  • 例如：
public class Student{
	 String name;
	  //构造方法
	 public Student(){
	 }
	 public Student(String name){
		this.name=name;
	 }
	  //普通方法
	 public void  show(){
	 }
}

# *7、this关键字

a、 代表当前类，可以调用属性和方法 语法： this.属性 this.方法() b、 区分同名的成员变量（属性）和局部变量 this.name=name; c、 调用当前类的构造方法(只能写在构造方法中的第一行非注释性语句) this(); this(23,"张三");

# *8、重载（overload）

特点： 同一个类中方法同名（方法名）不同参（参数个数，位置，顺序）与返回类型无关 什么时候用重载： 功能相同或类似 好处： 方便记忆

# 9、成员变量 位置：在类里边方法外面，并且具有默认值，用来描述类里边有什么

  • 作用域： 整个类里边都可以使用
  • 内存中位置： 在堆内存中，由于对象的存在，才在内存中存在；

# 10、局部变量 位置：在类的方法里边，必须赋初始值，用来临时存储数据

  • 作用域： 当前方法中都能使用
  • 内存中位置： 在栈内存中

# 11、值类型和引用类型

值类型： 值类型（int/double/short/byte/enum/long/float/char/boolean）保存在栈中 引用类型： 引用类型（String/类/数组/接口/Object）保存在堆中，但是会在栈中留下地址/引用 值传递： 如果方法的参数是值类型就是值传递 引用传递： 如果方法的参数是引用类型就是引用传递 区别： 值传递传本身（实际值），引用传递传引用（内存地址）

# 12、特殊的String类型

String是一个类，所以他是引用类型，但是在传递数据的时候却是 值传递 原因： String其实是一个char类型数组，由于数组长度固定，所以被赋值时需要重新创建数组来保存新数据， 新创建的数组和原来的数组没有关系，所以呈现值传递特点。

# *13、堆和栈

  • 堆： 由程序员分配，若程序员不释放，程序结束时由GC（垃圾回收器）回收。（对象=null）
  • 栈： 由操作系统自动分配存放数值，自动释放

# 14、对象数组

对象数组 中每一个元素都是一个对象，元素的值是对象的引用（内存地址） 语法： 类名[] 数组名 =new 类名[5];

Student[] stus = new Student[5];
1

使用：

  • a、给数组赋值
for(int i=0;i<stus.length;i++){

	stus[i].name=input.next();
	System.out.print("请输入"+stus[i.name+的龄：");
	stus[i].age=input.nextInt();
}
