# 为什么需要事务

为了保证数据库的完整性（完整性=准确性+可靠性）。

# 什么是事务

事务(TRANSACTION)是作为单个逻辑工作单元执行的一系列操作

# 事务必须具备以下四个属性并解释

简称ACID 属性

名称 意义
原子性（Atomicity） 事务是一个整体，不能分开
一致性（Consistency） 执行事务过程中要么都成功，要么都失败
隔离性（Isolation） 事务和事务之间互相不影响
持久性（Durability） 事务执行成功对数据的修改是永久保存的

# 如何使用事务

	START TRANSACTION / begin ; 开始事务
	commit; 提交事务
	rollback;回滚事务

# 什么是索引

索引是一种有效组合数据的方式，为快速查找到指定记录

# 索引作用

优点: 大大提高数据库的检索速度 改善数据库性能 缺点： 1、创建索引和维护索引要耗费时间，并且随着数据量的增加所耗费的时间也会增加 2、索引也需要占空间，我们知道数据表中的数据也会有最大上线设置的，如果我们有大量的索引，索引文件可能会比数据文件更快达到上线值 3、当对表中的数据进行增加、删除、修改时，索引也需要动态的维护，降低了数据的维护速度。

# 常用索引类型

# 普通索引
ALTER TABLE `table_name` ADD INDEX index_name ( `column` );
1
# 唯一索引
ALTER TABLE `table_name` ADD UNIQUE index_name ( `column` );
1
# 主键索引
ALTER TABLE `table_name` ADD PRIMARY KEY index_name ( `column` );
1
# 外键索引
ALTER TABLE 表名 ADD CONSTRAINT 外键名  FOREIGN KEY（外键字段）	REFERENCES 关联表名（关联字段）;
1
# 复合索引
ALTER TABLE `table_name` ADD INDEX index_name ( `column1`, `column2`, `column3` );
1
# 全文索引
ALTER TABLE `table_name` ADD FULLTEXT index_name ( `column`);
1

注意:只有在MyISAM引擎上才能使用，只能在CHAR,VARCHAR,TEXT类型字段上使用全文索引，就是在一堆文字中，通过其中的某个关键字等，就能找到该字段所属的记录行 空间索引 空间索引是对空间数据类型的字段建立的索引，MySQL中的空间数据类型有四种，GEOMETRY、POINT、LINESTRING、POLYGON。

# 创建索引的指导原则

  • 按照下列标准选择建立索引的列
    • 频繁搜索的列
    • 经常用作查询选择的列
    • 经常排序、分组的列
    • 经常用作连接的列（主键/外键）
  • 请不要使用下面的列创建索引
    • 仅包含几个不同值的列
    • 表中仅包含几行

# 索引相关经验

1、查询时减少使用* 返回全部列，不要返回不需要的列 2、索引应该尽量小，在字节数小的列上建立索引 3、WHERE子句中有多个条件表达式时，包含索引列的表达式应置于其他条件表达式之前 4、避免在ORDER BY子句中使用表达式

# 创建索引的语法

	CREATE [UNIQUE|FULLTEXT|SPATIAL] INDEX index_name
	ON table_name (column_name[length]…);
	例如：create  index index_phone on student(phone);

# 使用mysqldump命令备份数据库

	mysqldump [选项参数] –u 用户名 –h 主机名或ip -p [密码] 数据库名称 [表名,表名] > 文件路径和名称
	选项参数：
	-t 只导出数据，不包含create 语句    mysqldump -u root -p -t myschool > D:\backup\myschool_20170827.sql
	-c 每个insert语句带有列名  mysqldump -u root -p -c myschool > D:\backup\myschool_20170827.sql
	-d 只导出表结构			mysqldump -u root -p -d myschool > D:\backup\myschool_20170827.sql
	-w 查询条件	mysqldump -u root -p -w "gradeid=1" myschool student > D:\backup\myschool_20170827.sql
